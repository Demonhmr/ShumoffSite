# Handoff: Инструкция по передаче проекта Shumoff v2 исполнителям

Привет! Это сводный документ для команды дизайна и разработки. Здесь собраны все готовые ТЗ, структуры, тексты и куски кода, которые мы подготовили на этапе проектирования. Проект разбит на спринты (см. `План реализации.md`), и ваша задача — перенести эту базу в визуальный дизайн и функциональный сайт на WordPress.

---

## 🎨 Для UI/UX Дизайнера (Figma)

Твоя задача — отрисовать макеты, опираясь на готовую структуру и маркетинговые смыслы.

**С чего начать:**
1. Прочитай **`ТЗ на дизайн-концепцию.md`** — там указана стилистика, цвета, шрифты и требования к карточкам (Услуги, Кейсы, Отзывы).
2. Открой **`Техническое задание.md`** и **`Структура первого экрана.md`** — это твой каркас для отрисовки Главной страницы. Все тексты (УТП, буллиты, заголовки) уже написаны, используй их в макетах.
3. Изучи **`ui-blocks.html`** и **`hero-section.html`** — это готовая верстка конверсионных блоков. Ты можешь взять их за основу для дизайна (особенно блок "До/После" и таблицу цен).
4. Для понимания того, как будет выглядеть калькулятор, посмотри **`Квиз-калькулятор.md`**.

**Ожидаемый результат первого этапа:** UI Kit и макет Главной страницы (десктоп + мобилка).

---

## 👨‍💻 Для Web-разработчика (WordPress + Bricks Builder)

Твоя задача — развернуть сайт, настроить базу данных и перенести дизайн в Bricks Builder. Половина технической работы (регистрация CPT, таксономий и ACF) уже написана за тебя.

**С чего начать:**
1. Изучи **`ТЕХНИЧЕСКОЕ ЗАДАНИЕ полное.md`** — там описан стек плагинов (Bricks, ACF Pro, Rank Math, Fluent Forms), требования к безопасности и скорости.
2. Установи базовый плагин **`wp-content/plugins/shumoff-core/shumoff-core.php`** — в нем уже зарегистрированы все нужные Custom Post Types (Услуги, Автомобили, Кейсы, Отзывы), Таксономии (марки авто) и все кастомные поля ACF. В нем же реализована автоматическая генерация микроразметки FAQPage.
3. Ознакомься с **`SEO-структура и мета-теги.md`** — там прописаны правила для ЧПУ (URL) и шаблоны мета-тегов для Rank Math.
4. В файлах **`hero-section.html`** и **`ui-blocks.html`** лежат готовые HTML/CSS компоненты первого экрана и конверсионных блоков. Можешь перенести их в Bricks через элемент "Custom Code" или собрать нативных компонентах конструктора.
5. Настрой формы так, чтобы данные отправлялись в Telegram (ТЗ описано в `Квиз-калькулятор.md`).

## ✍️ Для Контент-менеджера / Копирайтера

Ваша задача — наполнить сайт так, чтобы он сразу начал получать SEO-трафик.

**С чего начать:**
1. Откройте **`Контентный план запуска Shumoff v2.md`** — там список первых 20 машин, 6 услуг и 30 кейсов, которые нужно залить на сайт до релиза. Там же лежит жесткий шаблон для оформления Кейса.
2. **`Статья - Какую шумоизоляцию выбрать.md`** — первая готовая статья для раздела "Блог". Ее можно публиковать.
3. При настройке форм не забудьте разместить текст из **`Политика конфиденциальности.md`** на отдельной странице и дать на нее ссылку в чекбоксах согласия.
4. Инструкция по перелинковке и SEO-шаблонам лежит в файле **`SEO-структура и мета-теги.md`**.
